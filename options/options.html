📖 快速开始

  1. 配置域名 - 在"域名配置"中添加允许录制的域名（如 localhost、*.example.com）
  2. 开始录制 - 点击扩展图标，选择"🔴 录制"
  3. 进行操作 - 在当前页面正常使用功能（访问接口、填写表单等）
  4. 停止录制 - 点击"⏹ 停止"保存记录
  5. 查看/导出 - 在记录管理中查看详情，或导出为 JSON/HAR 格式

🎯 界面说明

顶部控制栏

包含录制按钮（🔴 录制/⏹ 停止）和设置按钮（⚙️）。录制时按钮显示红色呼吸灯效果。

状态栏

左侧显示录制状态（● 就绪/录制中/回放中），右侧显示域名配置状态（● 已配置/未配置）。录制/回放时状态点会显示呼吸灯。

会话列表

显示所有会话，支持按类型筛选（全部/📹 录制/📥 导入）和搜索。点击 👁️ 查看详情，点击 ▶️ 回放。

会话标记

📹 表示本地录制的数据，📥 表示导入的外部数据。导入数据的时间显示为导入时间，标题使用文件名。

⚙️ 功能说明

网络请求捕获

自动捕获 XMLHttpRequest、Fetch 和 WebSocket 请求，包括请求头、请求体、响应数据和状态码。支持跨域请求捕获。

存储数据捕获

捕获 LocalStorage 和 SessionStorage 的快照，每隔 5 秒自动捕获一次。回放时可选择恢复这些数据。

域名配置

支持三种模式：
• 仅录制 - 只支持录制功能
• 仅回放 - 只支持回放功能
• 全部 - 支持录制和回放
支持通配符匹配（如 *.example.com）。

数据导出

JSON 格式 - 包含完整数据，适合备份和恢复
HAR 格式 - 兼容 Chrome DevTools、Postman、Charles 等工具

请求回放

拦截页面请求并返回录制的响应数据。支持：
• HTTP 请求回放（XHR/Fetch）
• WebSocket 消息按顺序推送
• LocalStorage/SessionStorage 数据恢复
• 未匹配的请求正常发出（混合模式）

数据导入

支持导入 JSON 和 HAR 格式文件。导入的数据：
• 标记为 📥
• 标题使用文件名（去掉扩展名）
• 时间显示为导入时间
• 可以像录制数据一样查看和回放

📤 JSON 导出功能

JSON 格式是 WebRecorder 的主要导出格式，支持丰富的自定义选项：

自定义导出内容

灵活选择需要导出的数据类型：
• 网络请求 - HTTP/XHR/Fetch/WebSocket 请求详情
• 存储快照 - LocalStorage 和 SessionStorage 数据
• 会话信息 - 时间、标题、域名等元数据

多种过滤方式

精确筛选需要导出的请求：
• 按类型过滤 - 仅 XHR、Fetch 或 WebSocket
• 按状态码过滤 - 仅成功请求(2xx)或失败请求
• URL 路径过滤 - 支持通配符匹配（如 /api/*）

格式选项

根据需求选择合适的格式：
• 美化格式 - 带缩进和换行，便于阅读和调试
• 压缩格式 - 去除空白字符，文件体积更小

导入回放

导出的 JSON 文件可以重新导入 WebRecorder：
• 在任意页面回放录制的请求
• 恢复存储数据到原始状态
• 团队成员共享和协作调试

🔧 HAR 格式简介

HAR（HTTP Archive）是标准的网络请求记录格式，仅包含 HTTP 请求信息：

  • 兼容 Chrome DevTools、Postman、Charles 等专业工具分析
  • 标准的 HTTP Archive 格式，可用于网络调试和性能分析
  • 仅包含 HTTP 请求数据（XHR/Fetch），不包含 WebSocket 和存储快照

⚠️ 注意事项

  • CSP 限制 - 某些网站设置了严格的内容安全策略，可能阻止脚本注入
  • 跨域限制 - 捕获其他域名的请求时，某些信息可能不完整
  • 性能影响 - 录制大量数据可能影响页面性能，建议定期清理旧记录
  • WebSocket 限制 - WebSocket 消息仅支持文本，二进制数据会被忽略
  • 回放精度 - 动态生成的 ID 可能无法完全匹配，依赖路径模式匹配
  • 扩展页面 - chrome-extension:// 页面支持查看会话列表，但不可录制/回放

💡 使用技巧

  • 过滤规则 - 在设置中添加 URL 或 LocalStorage 过滤规则，排除不需要的数据
  • 批量管理 - 在记录管理页面可以批量删除旧记录，释放存储空间
  • 团队协作 - 导出 JSON 文件分享给团队成员，他们可以导入后查看和回放
  • 问题排查 - 录制用户遇到的问题场景，导出 HAR 发给后端分析
